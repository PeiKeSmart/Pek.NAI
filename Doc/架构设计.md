## 3. NewLife.AI 基础库详细架构

### 3.1 服务商注册体系（AiClientRegistry）

通过**特性反射**自动发现和注册所有 AI 客户端，无需手动配置：

```csharp
// 客户端类声明（一个类可有多个 [AiClient] 特性）
[AiClient("openai", "OpenAI", "https://api.openai.com/v1")]
[AiClient("azure-openai", "Azure OpenAI", "https://{resource}.openai.azure.com")]
[AiClientModel("gpt-4o", Vision = true, FunctionCalling = true)]
[AiClientModel("o3", Thinking = true)]
public class OpenAiChatClient : IChatClient { ... }

// 查找并创建客户端
var descriptor = AiClientRegistry.Default.Find("openai");
var client = descriptor.CreateClient(endpoint, apiKey, model);
```

`AiClientRegistry` 在程序启动时扫描所有程序集，收集 `[AiClient]` 特性，构建 `AiClientDescriptor` 字典。`AiClientDescriptor` 包含：Code、DisplayName、DefaultEndpoint、Protocol、Factory 委托。

当前覆盖的服务商（通过 `OpenAiChatClient` 多特性机制）：

| 客户端类 | 覆盖服务商（部分） |
|---------|----------------|
| OpenAiChatClient | OpenAI、DeepSeek、Kimi、豆包、通义千问、智谱、百度、讯飞、腾讯混元、阶跃星辰、01.AI、Groq、Together、HuggingFace、NvidiaNIM、DeepInfra、CloudflareAI 等 ~40 个 |
| AzureAIChatClient | Azure OpenAI（部署名称 URL + api-key 认证） |
| BedrockChatClient | AWS Bedrock Converse API（SigV4 签名认证） |
| AnthropicChatClient | Anthropic（Claude 系列）|
| GeminiChatClient | Google Gemini 系列 |
| OllamaChatClient | Ollama（本地模型）|
| DashScopeChatClient | 阿里百炼专用客户端 |
| NewLifeAiChatClient | 级联 NewLife.AI 实例 |

### 3.2 ChatClientBuilder 中间件管道

```csharp
// 组装对话处理管道（类似 ASP.NET Core 的 IApplicationBuilder）
var client = rawProviderClient
    .AsBuilder()
    .UseFilters(new LoggingFilter(), new RateLimitFilter())  // IChatFilter 洋葱圈
    .UseTools(toolRegistry)                                   // 工具调用循环
    .Build();

// 管道执行顺序（请求方向）
// Filter1.Before  Filter2.Before  ToolClient  rawClient
// 返回方向:  rawClient  ToolClient  Filter2.After  Filter1.After
```

关键中间件类：

| 类 | 职责 |
|----|------|
| FilteredChatClient | 执行 IChatFilter 链（洋葱圈） |
| ToolChatClient | 自动多轮工具调用，直到模型停止请求工具 |
| DelegatingChatClient | 装饰器基类，子类只需覆盖需要的方法 |

### 3.3 客户端使用方式

`IChatClient` 支持四种使用模式，按场景选择：

#### 模式一：直接 `new`（最简，非 Web 场景）

无需任何注册或 DI 容器，直接实例化客户端：

```csharp
// 直接构造（apiKey, model?, endpoint?）
var client = new DashScopeChatClient("sk-xxx", "qwen3.5-flash");

// 可选：替换底层 HttpClient（设置代理、超时等）
client.HttpClient = new HttpClient { Timeout = TimeSpan.FromSeconds(30) };

var reply = await client.ChatAsync("你好");
```

各服务商便捷构造签名：

| Client 类 | apiKey | 说明 |
|-----------|--------|------|
| `OpenAIChatClient` | `String` | OpenAI 及全部兼容服务商 |
| `DashScopeChatClient` | `String` | 阿里百炼 |
| `AnthropicChatClient` | `String` | Claude |
| `GeminiChatClient` | `String` | Google Gemini |
| `AzureAIChatClient` | `String` | Azure OpenAI（部署名称作为 model） |
| `BedrockChatClient` | `String` | AWS Bedrock（accessKeyId，secretAccessKey 作为 apiKey + organization） |
| `OllamaChatClient` | `String?` | 本地部署可传 null |
| `NewLifeAiChatClient` | `String` | 级联 NewLife.AI 实例 |

#### 模式二：AiClientRegistry（配置驱动 / 多服务商切换）

适合从数据库或配置文件读取服务商信息、运行时动态切换的场景：

```csharp
// 通过服务商编码 + 选项创建
var client = AiClientRegistry.Default.CreateClient(
    "DashScope",
    new AiClientOptions { ApiKey = "sk-xxx", Model = "qwen3.5-flash" });

// 便捷重载：直接传字符串
var client = AiClientRegistry.Default.CreateClient("DashScope", "sk-xxx", "qwen3.5-flash");

// 从 AiClientOptions（含 Code 属性）创建——适合序列化读取
var opts = new AiClientOptions { Code = "OpenAI", ApiKey = "sk-xxx", Model = "gpt-4o" };
var client = AiClientRegistry.Default.CreateClient(opts.Code!, opts);
```

#### 模式三：ChatClientBuilder（中间件管道）

MEAI 风格，先设置服务商，再链式叠加过滤器、工具调用等中间件：

```csharp
var client = new ChatClientBuilder()
    .UseDashScope("sk-xxx", "qwen3.5-flash")   // 或 UseOpenAI / UseAnthropic / UseGemini / UseOllama / UseNewLifeAI
    .UseFilters(new LoggingFilter())              // 日志 / 审计
    .UseTools(toolRegistry)                       // 自动多轮 Function Calling
    .Build();
```

所有 Use* 方法签名统一为 `(apiKey, model=null, endpoint=null)`，Ollama 的 `apiKey` 可为 null。

#### 模式四：依赖注入（ASP.NET Core Web 应用）

```csharp
// ── 单服务商注册 ─────────────────────────────────────────
// 服务商专属注册（推荐，最简洁）
services.AddDashScope("sk-xxx", "qwen3.5-flash");
// 注入：IChatClient

// 通用配置驱动注册
services.AddAiClient(opts =>
{
    opts.Code   = "DashScope";
    opts.ApiKey = "sk-xxx";
    opts.Model  = "qwen3.5-flash";
});

// ── 多服务商注册（.NET 8+，Keyed Services）───────────────
services.AddKeyedDashScope("fast",   "sk-xxx", "qwen3.5-flash");
services.AddKeyedOpenAI  ("strong", "sk-xxx", "gpt-4o");
// 注入：[FromKeyedServices("fast")] IChatClient client
```

可用的专属注册方法（`NewLife.AI.Extensions`）：

| 单例注册 | Keyed 注册（.NET 8+） |
|---------|--------------------|
| `AddOpenAI(apiKey, model?, endpoint?)` | `AddKeyedOpenAI(key, apiKey, ...)` |
| `AddDashScope(...)` | `AddKeyedDashScope(...)` |
| `AddAnthropic(...)` | `AddKeyedAnthropic(...)` |
| `AddGemini(...)` | `AddKeyedGemini(...)` || `AddAzureAI(apiKey, model?, endpoint?)` | `AddKeyedAzureAI(key, apiKey, ...)` |
| `AddBedrock(accessKeyId, secretAccessKey, model?, region?)` | `AddKeyedBedrock(key, accessKeyId, ...)` || `AddOllama(apiKey?, ...)` | `AddKeyedOllama(key, apiKey?, ...)` |
| `AddNewLifeAI(...)` | `AddKeyedNewLifeAI(...)` |

#### 场景对应表

| 场景 | 推荐模式 |
|------|---------|
| 控制台工具 / 脚本 | 模式一（直接 new） |
| 服务商动态切换、配置数据库驱动 | 模式二（Registry） |
| 需要过滤器 / 工具调用管道 | 模式三（Builder） |
| ASP.NET Core Web 应用 | 模式四（DI）|
| 以上 + 多服务商并存 | 模式四 + Keyed Services（.NET 8+）|

### 3.4 IChatFilter 过滤器接口

```csharp
public interface IChatFilter
{
    // 请求前（可修改 options）
    Task OnRequestAsync(FilterContext context);

    // 流式输出完成后（可分析全量 response）
    Task OnStreamCompletedAsync(FilterContext context);
}
```

- `FilterContext` 携带请求参数、响应结果、自定义状态（键值字典）。
- 多个 Filter 按注册顺序形成洋葱圈，before 正序 / after 逆序执行。

```
ToolRegistry                     # 工具注册中心
```

### 4.4 请求处理流程

```
前端发送消息
    
    
MessagesController.PostMessage()
    
    
MessageRateLimiter.CheckAsync()        # 速率检查（20 次/分钟）
    
    
ChatApplicationService.SendAsync()
     1. 保存用户消息（ChatMessage，Role=user）
     2. 构建上下文（最近 N 轮消息）
     3. IChatPipeline.ExecuteAsync()
            AiClientRegistry 获取客户端
            ChatClientBuilder 组装管道
               FilteredChatClient（过滤器）
               ToolChatClient（工具调用循环）
            调用 IChatClient.GetStreamingResponseAsync()
     4. 推送 SSE 事件到前端
           thinking_delta / tool_call_* / content_delta / message_done
     5. 保存 AI 消息 + 更新 Token 统计
```

### 5.3 附件存储

附件不使用独立实体，而是复用 **NewLife.Cube 的 Attachment 实体**：

- 上传时 Category = "ChatAI"，Key = MessageId（发送前为 "temp_{UserId}"）。
- ChatMessage.Attachments 存储附件 ID 数组的 JSON，如 [123456, 789012]。
- 下载 / 预览通过 AttachmentsController 代理到魔方附件接口。

### 5.4 使用量统计

> 当前无独立 UsageRecord 表，使用量从 Conversation 实体聚合计算。

- 每次 AI 回复完成后，将 Token 消耗累计到 Conversation.TotalTokens 等字段。
- UsageApiController 通过聚合 Conversation 实体提供统计接口。
- /api/usage/summary：返回用户累计会话数、消息数、Token 总量。
- /api/usage/daily：按日聚合 Conversation 记录，支持日期范围查询。

## 7. SSE 流式协议规范

### 7.1 协议概述

POST /api/conversations/{id}/messages 以 SSE 格式返回响应：

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

每条事件格式：

```
data: {JSON}\n\n
```

### 7.2 请求格式

```json
{
  "content": "用户消息内容",
  "thinkingMode": 0,
  "attachmentIds": [123456]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| content | String | 消息文本 |
| thinkingMode | Int32 | 0=自动, 1=思考, 2=快速 |
| attachmentIds | Int64[] | 附件 ID 列表 |

### 7.3 SSE 事件类型

| 事件类型 | 触发时机 | 关键字段 |
|---------|---------|---------|
| message_start | 流开始 | messageId, model, thinkingMode |
| thinking_delta | 思考内容增量 | content |
| thinking_done | 思考完成 | thinkingTime（ms） |
| content_delta | 回答内容增量 | content |
| tool_call_start | 工具调用开始 | toolCallId, name, arguments |
| tool_call_done | 工具调用完成 | toolCallId, result, success |
| tool_call_error | 工具调用失败 | toolCallId, error |
| message_done | 全部完成 | usage（promptTokens, completionTokens, totalTokens） |
| error | 错误中断 | code, message |

### 7.4 典型流示例（交错思考 + 工具调用）

```
data: {"type":"message_start","messageId":5004,"model":"gpt-4o","thinkingMode":1}

data: {"type":"thinking_delta","content":"用户想知道北京天气，我需要调用天气工具..."}

data: {"type":"thinking_done","thinkingTime":800}

data: {"type":"tool_call_start","toolCallId":"call_001","name":"get_weather","arguments":"{\"city\":\"北京\"}"}

data: {"type":"tool_call_done","toolCallId":"call_001","result":"{\"temp\":25,\"weather\":\"晴\"}","success":true}

data: {"type":"thinking_delta","content":"天气数据已获取，我来组织回复..."}

data: {"type":"thinking_done","thinkingTime":500}

data: {"type":"content_delta","content":"北京今天晴好，气温 25C。"}

data: {"type":"message_done","usage":{"promptTokens":80,"completionTokens":60,"totalTokens":140}}
```
